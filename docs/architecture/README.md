## ADRの作成ガイドライン

1. **ファイル命名規則**
   - 形式: `adr-{番号}-{簡潔な説明}.md`
   - 例: `adr-001-static-site-generation.md`

2. **必須セクション**
   - ステータス
   - 背景
   - 決定事項
   - 代替案の検討
   - 影響
   - 対策
   - 注意点

3. **ステータスの種類**
   - 提案中（Proposed）
   - 採用済み（Accepted）
   - 廃止（Deprecated）
   - 置換（Superseded）

4. **更新ルール**
   - 既存のADRは変更しない
   - 決定事項の変更は新しいADRとして記録
   - 古いADRは「置換」としてマーク
